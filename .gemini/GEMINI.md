# 나만의레시피 - 워크스페이스 규칙

---

## 1. 프로젝트 목표 (7단계 확장 메뉴)

사용자가 단계별로 성장하며 사용할 수 있는 **올인원 관리 시스템**을 구축합니다.

| 단계 | 메뉴 | 기능 |
|------|------|------|
| 1 | 🍳 **레시피 저장소** | 사진, 재료, 조리법, 셰프 팁 기록 및 검색 |
| 2 | 🛒 **장보기/재료 관리** | 시장 정보, 단가 비교, 체크리스트 관리 |
| 3 | 🏠 **창업 준비 로그** | 입지 분석, 인테리어 아이디어, 행정 절차 기록 |
| 4 | 💰 **세무/회계 노트** | 일일 지출 기록 및 상담 메모 |
| 5 | 🤝 **거래처 관리** | 식자재 업체 연락처 및 거래 조건 관리 |
| 6 | 📅 **예약 시스템** | 예약 현황 확인, 알림, 캘린더 뷰 |
| 7 | 👥 **고객 관리** | 단골 고객 성향, 선호도, 특이사항 기록 |

## 2. 개발 및 로직 원칙 (강력한 오류 방지)

> [!CAUTION]
> 이 원칙들은 반드시 준수해야 합니다!

### 로직 우선 검토
- 코드를 작성하기 전, **'데이터 흐름'을 먼저 설계하고 설명**해야 한다.

### 방어적 코딩
- 모든 입력창(Input)에는 **유효성 검사(Validation)**를 포함하여 잘못된 데이터가 들어가지 않도록 한다.

### 증분 수정 원칙 (UI 보호)
- 기능을 수정하거나 추가할 때, **기존의 전체 UI 구조를 유지**해야 한다.
- **Layout을 통째로 바꾸는 행위는 엄격히 금지**하며, 필요한 부분만 정교하게 수정한다.

### 파일 상단 정보
- 생성/수정하는 모든 파일 최상단에 다음을 반드시 주석으로 명시한다:
```typescript
// Path: [파일경로]
// Description: [기능설명]
```

## 3. 디자인 원칙 (UX/UI 일관성)

### 테마 유지
- **Warm Wood & Olive Green** 포인트의 프리미엄 디자인을 모든 페이지에 일관되게 적용한다.

### 컴포넌트 재사용
- 버튼, 입력창, 카드 레이아웃은 **공통 컴포넌트**로 만들어 UI가 페이지마다 달라지는 것을 방지한다.

### 반응형 검증
- 스마트폰(iOS/Android), 태블릿, PC 브라우저에서 레이아웃이 깨지지 않는지 **코딩 단계에서 스스로 체크**한다.

### 컬러 팔레트
- **Primary (Olive Green)**: 자연스럽고 건강한 느낌
- **Secondary (Warm Wood)**: 따뜻하고 아늑한 분위기
- **Background (Cream)**: 깔끔하고 정갈한 배경

## 4. 에이전트 수행 가이드 (기획자 협업)

> [!IMPORTANT]
> 이 프로젝트의 사용자는 코딩을 모르는 기획자입니다.

### 선검토 후코딩
- 작업을 시작하기 전 **"현재 로직에서 예상되는 문제점"**을 먼저 나열하고, 기획자에게 확인을 받은 뒤 코딩을 시작한다.

### 용어 순화
- 기술적 에러(예: 404, Null Pointer 등)를 사용자에게 노출하지 않고, **친절한 한국어 안내 메시지**로 처리한다.

### 변경 사항 요약
- 수정이 완료된 후에는 **'무엇이 바뀌었는지'와 '어떤 오류가 수정되었는지'**를 기획적 관점에서 **3줄 요약**한다.

## 5. 기술 사양 (Tech Stack)

| 항목 | 기술 | 비고 |
|------|------|------|
| Framework | **Next.js (App Router)** | 최신 기술, 확장성 좋음 |
| Styling | **Tailwind CSS** | 일관된 디자인 시스템 적용 |
| Icons | **Lucide React** | 직관적인 아이콘 |
| Storage | **Supabase** | 클라우드 DB + 이미지 저장, 모든 기기 동기화 |
| State | **'use client' 컴포넌트** | 상태 관리를 직관적으로 처리하여 버그 최소화 |
